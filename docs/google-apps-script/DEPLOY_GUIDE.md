# Apps Script 배포 가이드

## 1. 구글 시트 + Apps Script 설정

1. **구글 시트 새로 만들기**
   - 브라우저에서 [sheets.new](https://sheets.new) 열기
   - 시트 이름: 아무거나 (예: "소싱 위시리스트")
   - Apps Script가 자동으로 `sourcing` 이름의 시트 탭을 만들어줌

2. **Apps Script 열기**
   - 상단 메뉴 → **확장 프로그램** → **Apps Script**
   - 새 탭으로 편집기가 열림

3. **Code.gs 붙여넣기**
   - 편집기에 있는 기존 내용(`function myFunction() {}`) 전체 선택 후 삭제
   - `Code.gs` 파일 내용 전체를 붙여넣기
   - **저장** (Ctrl+S 또는 💾 버튼)

## 2. Web App 배포

1. 우측 상단 **배포** 버튼 → **새 배포**

2. 배포 설정:
   - 유형: **웹 앱**
   - 설명: 소싱 위시리스트 API (임의)
   - 다음 사용자로 실행: **나** (본인 Google 계정)
   - 액세스 권한: **모든 사람** ← 반드시 이걸로 선택

3. **배포** 클릭 → Google 계정 권한 요청 팝업 → **허용**

4. 배포 완료 후 **웹 앱 URL** 복사
   - 형식: `https://script.google.com/macros/s/AKfycb.../exec`
   - 이 URL이 webhook URL (비밀번호 역할) — 외부에 노출하지 말 것

## 3. 동작 확인

브라우저 주소창에 아래 URL 입력해서 빈 배열 `[]`이 나오면 성공:
```
https://script.google.com/macros/s/[YOUR_ID]/exec?action=pending
```

## 4. 코드 수정 후 재배포

코드를 수정했을 때는 **새 배포** 대신:
- **배포** → **배포 관리** → 연필(수정) 아이콘 → **버전: 새 버전** → 배포

> ⚠️ "새 배포"를 다시 하면 URL이 바뀜 — 기존 단축어와 익스텐션 설정을 모두 업데이트해야 함.
> 수정 시에는 반드시 기존 배포를 "업데이트"할 것.

## 5. 배포 완료 후 다음 단계

- iOS 단축어 셋업: `SHORTCUT_GUIDE.md` 참조
- 익스텐션 설정 모달에 webhook URL 입력

## 6. 소싱 원장 (`ledger` 시트) 초기 설정

Code.gs 재배포 후 구글 시트를 열면 **소싱 원장** 메뉴가 상단에 생깁니다.

### 6-1. `ledger` 시트 자동 생성 확인
- Step 4 임시저장을 한 번 완료하면 `ledger` 시트가 자동 생성됨
- 1행 헤더: `현황 / 1688링크 / 상품명 / 색상 / 수량 / 중국원가 / 공급가 / 판매가 / END ROAS`
- A열 드롭다운(제안중/통과/반려/수입완료)과 조건부 서식(통과=초록, 반려=빨강, 수입완료=회색) 자동 적용

### 6-2. END ROAS 수식 1회 입력
첫 번째 상품이 삽입된 후 **2행의 I열(END ROAS)**에 원하는 수식을 직접 입력.
이후 신규 행 삽입 시 해당 수식이 자동 복사됨.

예시 수식: `=H2/G2` (판매가÷공급가)

### 6-3. 필터 보기로 통과 상품만 보기
1. 시트 상단 **데이터 → 필터 만들기**
2. A열(현황) 필터 아이콘 클릭 → '통과'만 선택
3. **데이터 → 필터 보기 → 현재 필터를 필터 보기로 저장** → "통과만" 이름으로 저장

다음부터는 필터 보기 메뉴에서 "통과만" 클릭으로 바로 전환 가능.

### 6-4. 수입완료 상품 보관
1. 현황 열에 '수입완료' 입력
2. 상단 메뉴 **소싱 원장 → 수입완료 보관** 클릭
3. 해당 행이 `완료(보관)` 시트로 이동 → `ledger` 시트가 가벼워짐
