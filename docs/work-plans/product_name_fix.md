## 문제 원인

`scraper.py`의 상품명 추출이 잘못된 값을 가져오던 이유:

1. **`window.runParams` 미작동** — 1688 페이지에서 이 변수가 없거나 구조가 다름 → DOM 방식으로 폴백
2. **DOM h1 선택 오류** — h1 중 가장 긴 것을 선택하는데, 추천 상품/사이드바 영역의 h1을 잡음
3. **Chrome 자동번역** — 페이지가 이미 한국어로 번역된 상태에서 DOM을 읽으니 번역된 텍스트 추출됨

결과: 이미지/가격은 올바른 상품에서, 상품명은 엉뚱한 요소에서 가져오는 현상

## 수정 방법

상품명 추출 우선순위 변경:

### 1순위: `<script type="application/ld+json">` 파싱
- 번역 영향 없음, 항상 원문 중국어
- `JSON.parse()` → `name` 필드 추출

### 2순위: `document.title` 파싱
- 탭 제목 = "상품명 - 판매자 - 1688.com" 형식
- `" - "` 기준 첫 번째 토큰이 상품명
- 자동번역이 켜져 있어도 title 태그는 원문 유지되는 경우가 많음

### 3순위: `window.runParams`

### 4순위: DOM (추천/사이드바 영역 제외)
